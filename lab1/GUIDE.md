# Лаба 1 — Прямые и итерационные методы решения СЛАУ. Собственные значения

## Общая идея лабы

Вся лаба крутится вокруг одной вещи: **систем линейных уравнений** (СЛАУ). Это когда у тебя несколько уравнений с несколькими неизвестными, и все уравнения линейные (нет x^2, sin(x) и т.д.). В матричной записи: **A·x = b**.

Задачи 1-3 решают СЛАУ разными методами. Задачи 4-5 ищут **собственные значения** матрицы (числа λ, при которых A·v = λ·v).

## Задача 1 — LU-разложение (`task1.py`)

### Суть простыми словами

Представь, что матрицу A можно «разобрать» на две более простые матрицы: нижнюю треугольную L и верхнюю треугольную U, так что **A = L·U**. Зачем? Потому что решать треугольные системы элементарно — подставляешь сверху вниз (или снизу вверх).

### Математика

Метод Гаусса — это когда ты вычитаешь строки друг из друга, чтобы получить нули под диагональю. LU-разложение — это тот же Гаусс, но мы **запоминаем множители** (на что умножали строку перед вычитанием) в матрицу L.

**Выбор главного элемента**: перед каждым шагом мы ищем строку с самым большим по модулю элементом в текущем столбце и ставим её наверх (перестановка строк). Это нужно для **устойчивости** — чтобы не делить на маленькие числа и не копить ошибки округления. Поэтому формула не A = LU, а **PA = LU**, где P — матрица перестановок.

### Что делает код

1. **Строки 29-49**: Основной цикл по k. На каждом шаге:
   - Ищет строку с максимальным |a_ik| (выбор главного элемента)
   - Если нужно — меняет строки местами в U, L и perm
   - Вычисляет множители Гаусса μ = a_ik / a_kk и кладёт их в L
   - Вычитает строки: a_ij = a_ij - μ · a_kj

2. **Строки 79-87**: Решение СЛАУ через L и U:
   - Сначала Ly = Pb (прямой ход — сверху вниз, просто подставляем)
   - Потом Ux = y (обратный ход — снизу вверх)

3. **Строки 101-108**: Определитель: det(A) = (-1)^p · u_11 · u_22 · ... · u_nn, где p — число перестановок строк.

4. **Строки 114-129**: Обратная матрица: решаем СЛАУ A·x_j = e_j для каждого столбца единичной матрицы. Используем одно и то же LU.

### Что говорить на защите

- «Разбиваем матрицу на нижнюю и верхнюю треугольные — PA = LU»
- «Выбор главного элемента — чтобы не делить на ноль или малое число»
- «Множители Гаусса μ = a_ik / a_kk записываются в L»
- «Определитель — произведение диагонали U с учётом знака от перестановок»
- «Обратная матрица — решаем n СЛАУ с одним и тем же LU»

## Задача 2 — Метод прогонки (`task2.py`)

### Суть простыми словами

Когда матрица **трёхдиагональная** (ненулевые элементы только на главной диагонали и двух соседних), решать систему LU-разложением — как стрелять из пушки по воробьям. Метод прогонки (алгоритм Томаса) — это оптимизированный Гаусс специально для такой структуры. Работает за O(n) вместо O(n^3).

### Математика

Система: a_i·x_{i-1} + b_i·x_i + c_i·x_{i+1} = d_i

**Прямой ход** (вычисляем коэффициенты P и Q):
- P_1 = -c_1/b_1, Q_1 = d_1/b_1
- P_i = -c_i / (b_i + a_i·P_{i-1}), Q_i = (d_i - a_i·Q_{i-1}) / (b_i + a_i·P_{i-1})

**Обратный ход** (находим x):
- x_n = Q_n (потому что P_n = 0)
- x_i = P_i·x_{i+1} + Q_i

Условие устойчивости (формула 1.8): |b_i| >= |a_i| + |c_i| — **диагональное преобладание**.

### Что делает код

1. **Строки 16-19**: Три массива a, b, c — три диагонали; d — правая часть.
2. **Строки 28-33**: Проверка диагонального преобладания.
3. **Строки 50-59**: Прямой ход — вычисление P и Q.
4. **Строки 70-73**: Обратный ход — вычисление x.
5. **Строки 82-89**: Проверка подстановкой — невязки должны быть ~0.

### Что говорить на защите

- «Прогонка — специализированный метод для трёхдиагональных систем, O(n) вместо O(n^3)»
- «Прямой ход считает коэффициенты P_i и Q_i от первого уравнения к последнему»
- «Обратный ход восстанавливает решение от x_n к x_1 по формуле x_i = P_i·x_{i+1} + Q_i»
- «Условие устойчивости — диагональное преобладание»

## Задача 3 — Метод простых итераций и Зейдель (`task3.py`)

### Суть простыми словами

Задачи 1 и 2 — **прямые** методы: за конечное число шагов получаем точный ответ (с точностью до округлений). Задача 3 — **итерационные** методы: начинаем с какого-то приближения и постепенно уточняем, пока разница между итерациями не станет меньше заданной точности ε.

### Математика

**Шаг 1**: Приводим систему A·x = b к виду **x = B·x + g** (формула 1.18):
- Делим каждое i-е уравнение на a_ii
- B_ij = -a_ij / a_ii (при i ≠ j), g_i = b_i / a_ii

**Условие сходимости**: ||B|| < 1 (норма матрицы B меньше единицы). Если это выполнено — оба метода сойдутся.

**Метод простых итераций (Якоби)**: x^{k+1} = B·x^k + g
- На каждом шаге вычисляем все новые x_i из **старых** значений x^k
- Оценка погрешности (1.20): ε^k = ||B|| / (1 - ||B||) · ||x^k - x^{k-1}||

**Метод Зейделя**: то же самое, но при вычислении x_i^{k+1} мы используем **уже обновлённые** x_1^{k+1}, ..., x_{i-1}^{k+1}.
- Сходится быстрее, потому что «свежая» информация используется сразу
- Оценка: ε^k = ||C|| / (1 - ||B||) · ||x^k - x^{k-1}||, где C — верхнетреугольная часть B

### Что делает код

1. **Строки 33-39**: Формируем B и g из A и b.
2. **Строки 42-43**: Считаем ||B||_∞ (максимальная строчная сумма модулей).
3. **Строки 64-80**: Метод Якоби — цикл x_new = g + B·x, пока оценка погрешности < ε.
4. **Строки 101-129**: Метод Зейделя — внутри цикла x[i] обновляется сразу (без x_new).
5. **Строка 136**: Сравнение: Зейдель сошёлся примерно в 4 раза быстрее.

### Что говорить на защите

- «Приводим систему к виду x = Bx + g делением на диагональные элементы»
- «Якоби — все компоненты обновляются из старых значений; Зейдель — из уже обновлённых»
- «Зейдель быстрее, потому что сразу использует свежую информацию»
- «Условие сходимости: ||B|| < 1 — обеспечивается диагональным преобладанием матрицы A»
- «Оценка погрешности через ||Δx|| и норму B — формулы 1.20 и замечание из §1.1.4»

## Задача 4 — Метод вращений Якоби (`task4.py`)

### Суть простыми словами

Дана **симметричная** матрица. Нужно найти её **собственные значения** λ и **собственные векторы** v — такие, что A·v = λ·v. Метод: поворачиваем матрицу так, чтобы внедиагональные элементы становились всё меньше, а на диагонали оставались собственные значения.

### Математика

На каждой итерации:
1. Находим максимальный по модулю внедиагональный элемент a_{pq}
2. Вычисляем угол поворота: φ = (1/2)·arctg(2·a_{pq} / (a_{pp} - a_{qq}))
3. Строим матрицу вращения U (единичная, но в позициях p,q стоят cos φ и sin φ)
4. Обновляем: A^{k+1} = U^T · A^k · U

**Почему это работает**: каждое вращение обнуляет выбранный элемент a_{pq}, а сумма квадратов внедиагональных элементов строго уменьшается. В пределе матрица становится диагональной — на диагонали собственные значения.

**Собственные векторы**: накапливаем произведение всех матриц вращения V = U_0 · U_1 · ... · U_k. Столбцы V — собственные векторы.

Критерий останова: t(A) = sqrt(Σ a_{ij}^2 для i≠j) < ε.

### Что делает код

1. **Строки 49-97**: Основной цикл:
   - Считает t(A) — норму внедиагональных элементов
   - Ищет максимальный |a_{pq}|
   - Считает угол φ
   - Обновляет A (формулы из §1.2.2)
   - Обновляет V (накапливает вращения)
2. **Строки 102-109**: Результат: λ_j = A[j][j], v_j — столбцы V.
3. **Строки 114-129**: Проверки: ||A·v - λ·v|| ≈ 0, ортогональность <v_i, v_j> ≈ 0.

### Что говорить на защите

- «Метод вращений — для симметричных матриц. Поворотами убираем внедиагональные элементы»
- «На каждой итерации обнуляем самый большой внедиагональный элемент»
- «Угол вращения: φ = (1/2)·arctg(2a_{pq} / (a_{pp} - a_{qq}))»
- «Собственные векторы накапливаются в произведении всех матриц вращения»
- «Проверяем: A·v_j = λ_j·v_j и ортогональность <v_i, v_j> = 0»

## Задача 5 — QR-разложение и QR-алгоритм (`task5.py`)

### Суть простыми словами

QR-алгоритм — самый мощный метод нахождения собственных значений. Работает для **любых** матриц (не только симметричных) и может находить **комплексные** собственные значения. Идея: раскладываем матрицу на Q (ортогональная) и R (верхняя треугольная), потом перемножаем в обратном порядке.

### Математика

**QR-разложение** (метод Хаусхолдера, формулы 1.29-1.30): любую матрицу можно представить как A = Q·R, где Q^T·Q = E (ортогональная), R — верхняя треугольная.

Матрица Хаусхолдера: H = E - 2·(v·v^T)/(v^T·v) — это «отражение», которое обнуляет все элементы в столбце ниже диагонали за один раз. Применяем последовательно для каждого столбца.

**QR-алгоритм**: A^k = Q_k · R_k, затем A^{k+1} = R_k · Q_k. Повторяем. Матрица A^k сходится к **квазитреугольной** (блочно-треугольной) форме:
- Вещественные λ — на диагонали
- Комплексные пары — в 2×2-блоках на диагонали

**Критерий останова**:
- Для столбца m: sqrt(Σ a_{im}^2, i>m) < ε → вещественное λ_m = a_{mm}
- Для 2×2-блока: |λ^k - λ^{k-1}| < ε → комплексная пара

### Что делает код

1. **Функция `qr_householder` (строки 18-49)**: QR-разложение:
   - Для каждого столбца k строит вектор Хаусхолдера v
   - Применяет отражение к R (обнуляет поддиагональные элементы)
   - Накапливает Q

2. **Функция `run_qr_algorithm` (строки 65-131)**: QR-алгоритм:
   - На каждой итерации: A = Q·R, потом A = R·Q
   - Проверяет критерий останова для каждого столбца и каждого 2×2-блока
   - Заканчивает, когда все λ «сошлись»

3. **Функция `extract_eigenvalues` (строки 139-168)**: Извлекает λ из финальной матрицы:
   - Если поддиагональный элемент мал → вещественное λ = a_{mm}
   - Если нет → 2×2-блок, решаем квадратное уравнение

4. **Строка 224**: Проверка: tr(A) = Σλ (сумма собственных значений = след матрицы).

### Что говорить на защите

- «QR-разложение методом Хаусхолдера: отражениями обнуляем поддиагональные элементы столбец за столбцом»
- «QR-алгоритм: разложили A = QR, перемножили R·Q — получили новую A. Повторяем»
- «Матрица сходится к квазитреугольной форме: вещественные λ на диагонали, комплексные пары в 2×2-блоках»
- «Проверяем: след матрицы = сумма собственных значений»
- «В отличие от метода Якоби (задача 4), QR-алгоритм работает для несимметричных матриц и находит комплексные λ»
