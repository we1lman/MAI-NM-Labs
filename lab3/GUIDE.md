## Задача 3.1 — Интерполяция Лагранжа и Ньютона (`task3_1.py`)

### Суть простыми словами

Дано 4 точки. Через любые 4 точки можно провести ровно один многочлен 3-й степени. Мы строим этот многочлен двумя способами (Лагранж и Ньютон) и вычисляем его значение в заданной точке X*.

### Вариант 4

- Функция: y = ctg(x)
- Набор а): узлы π/8, 2π/8, 3π/8, 4π/8
- Набор б): узлы π/8, 5π/16, 3π/8, π/2
- Точка интерполяции: X* = π/3

### Формула Лагранжа (3.5)

L_n(x) = Σ f_i · l_i(x), где l_i(x) = Π_{j≠i} (x - x_j)/(x_i - x_j)

По-простому: для каждой точки i строим «базисный многочлен» l_i(x), который равен 1 в точке x_i и 0 во всех остальных точках. Потом суммируем f_i · l_i(x).

```python
# Строки 20-28:
for i in range(n):
    li = 1.0
    for j in range(n):
        if j != i:
            li *= (x - X[j]) / (X[i] - X[j])   # l_i(x)
    result += F[i] * li                          # f_i · l_i(x)
```

### Формула Ньютона (3.8) с разделёнными разностями (3.7)

P_n(x) = f(x_0) + (x-x_0)·f(x_0,x_1) + (x-x_0)(x-x_1)·f(x_0,x_1,x_2) + ...

**Разделённые разности** — это аналог производных для дискретных данных:
- 0-й порядок: f(x_i) = f_i
- 1-й порядок: f(x_i, x_j) = (f_j - f_i) / (x_j - x_i)
- 2-й порядок: f(x_i, x_j, x_k) = (f(x_j, x_k) - f(x_i, x_j)) / (x_k - x_i)

```python
# Строки 40-44 — таблица разделённых разностей:
for j in range(1, n):
    for i in range(n - j):
        dd[i][j] = (dd[i+1][j-1] - dd[i][j-1]) / (X[i+j] - X[i])
```

**Ньютон vs Лагранж**: математически дают тот же самый многочлен. Но Ньютон удобнее — при добавлении новой точки не нужно всё пересчитывать.

### Что говорить на защите

- «Лагранж и Ньютон строят один и тот же интерполяционный многочлен, но разными способами»
- «Лагранж — через базисные полиномы l_i, каждый из которых равен 1 в своём узле и 0 в остальных»
- «Ньютон — через разделённые разности, удобнее при добавлении новых узлов»
- «Набор б) точнее, потому что X* = π/3 ближе к центру набора б)»

## Задача 3.2 — Кубический сплайн (`task3_2.py`)

### Суть простыми словами

Интерполяция одним многочленом высокой степени плохо работает при большом числе точек (многочлен «скачет» между узлами — эффект Рунге). Решение: на каждом отрезке [x_{i-1}, x_i] строим **свой кубический многочлен**, и «сшиваем» их на стыках.

### Математика (формула 3.11)

На каждом отрезке сплайн имеет вид:

S_i(x) = a_i + b_i·(x - x_{i-1}) + c_i·(x - x_{i-1})^2 + d_i·(x - x_{i-1})^3

**Условия «сшивки»** (непрерывность функции, 1-й и 2-й производных на стыках):
- S_i(x_i) = f_i — проходит через точки
- S_i(x_i) = S_{i+1}(x_i) — непрерывность
- S'_i(x_i) = S'_{i+1}(x_i) — гладкость
- S''_i(x_i) = S''_{i+1}(x_i) — гладкость кривизны

**Натуральный сплайн**: на концах S''(x_0) = 0, S''(x_n) = 0.

Из этих условий получается **трёхдиагональная система** (3.13) для коэффициентов c_i (кривизна). Решаем её прогонкой (как в задаче 1.2!), потом восстанавливаем a, b, d по формулам (3.14).

```python
# Строка 61 — правая часть системы для c_i:
rhs[j] = 3 * ((F[i] - F[i-1]) / h[i] - (F[i-1] - F[i-2]) / h[i-1])

# Строки 76-89 — прогонка (точно как в задаче 1.2)

# Строки 106-116 — восстановление a, b, d (формула 3.14):
a[i] = F[i-1]
b[i] = (F[i] - F[i-1]) / h[i] - h[i] * (c[i+1] + 2*c[i]) / 3
d[i] = (c[i+1] - c[i]) / (3 * h[i])
```

### Что говорить на защите

- «Сплайн — кусочно-кубический: на каждом отрезке свой кубический многочлен»
- «Сшивается по непрерывности функции, первой и второй производной»
- «Натуральный сплайн — S''=0 на концах»
- «Коэффициенты c_i находятся из трёхдиагональной системы, решаемой прогонкой»
- «Проверка: сплайн точно проходит через все узловые точки»

## Задача 3.3 — Метод наименьших квадратов (`task3_3.py`)

### Суть простыми словами

Интерполяция проходит **через все точки**. Но если данные с шумом — это плохо: кривая будет «дёргаться». МНК ищет **наиболее близкую** простую кривую (прямую или параболу), которая **не обязана** проходить через точки, но минимизирует суммарное отклонение.

### Математика (формулы 3.15-3.17)

Ищем F_n(x) = a_0 + a_1·x + ... + a_n·x^n, минимизирующий:

Φ = Σ [F_n(x_j) - y_j]^2 → min

Берём производные по каждому a_k и приравниваем к нулю. Получается **нормальная система МНК** (3.17):

Σ a_i · (Σ x_j^{k+i}) = Σ y_j · x_j^k, для k = 0, 1, ..., n

**Для прямой** (n=1, система 2×2):
```
(N+1)·a_0 + Σx·a_1 = Σy
Σx·a_0 + Σx^2·a_1 = Σxy
```

**Для параболы** (n=2, система 3×3):
```
(N+1)·a_0 + Σx·a_1 + Σx^2·a_2 = Σy
Σx·a_0 + Σx^2·a_1 + Σx^3·a_2 = Σxy
Σx^2·a_0 + Σx^3·a_1 + Σx^4·a_2 = Σx^2y
```

```python
# Строки 90-93 — система для прямой (формулы подставлены):
A1 = [[N+1, sum_x], [sum_x, sum_x2]]
b1 = [sum_y, sum_xy]

# Строки 128-133 — система для параболы:
A2 = [[N+1, sum_x, sum_x2], [sum_x, sum_x2, sum_x3], [sum_x2, sum_x3, sum_x4]]
b2 = [sum_y, sum_xy, sum_x2y]
```

Решаем системы (Крамер для 2×2, Гаусс для 3×3), получаем коэффициенты.

### Что говорить на защите

- «МНК минимизирует сумму квадратов отклонений — не проходит через точки, а ближе всего к ним»
- «Нормальная система: берём производные Φ по a_k, приравниваем к нулю»
- «Для прямой — система 2×2, для параболы — 3×3»
- «Φ_2 < Φ_1 — парабола аппроксимирует лучше, чем прямая»

## Задача 3.4 — Численное дифференцирование (`task3_4.py`)

### Суть простыми словами

Есть таблица значений функции в 5 точках. Нужно найти производные (первую и вторую) в точке X* = 0.2, **не зная формулу функции**.

### Формулы

**Первая производная 1-го порядка точности** (формула 3.18):

y'(x*) ≈ (y_{i+1} - y_i) / (x_{i+1} - x_i)

Это просто наклон секущей. «Левосторонняя» — по отрезку слева от X*, «правосторонняя» — по отрезку справа.

```python
dy_left = (Y[i+1] - Y[i]) / (X[i+1] - X[i])      # секущая слева
dy_right = (Y[i+2] - Y[i+1]) / (X[i+2] - X[i+1])  # секущая справа
```

**Первая производная 2-го порядка точности** (формула 3.20):

Используем три точки вместо двух — это как «средняя» производная. Для равномерной сетки совпадает с полусуммой левосторонней и правосторонней.

```python
dy_second = diff1 + second_diff * (2*x_star - X[i] - X[i+1])
# Для X*=0.2 (совпадает с x_2): = (dy_left + dy_right) / 2
```

**Вторая производная** (формула 3.21):

y''(x*) ≈ 2 · (f(x_i, x_{i+1}) - f(x_{i-1}, x_i)) / (x_{i+1} - x_{i-1})

Это разделённая разность 2-го порядка, умноженная на 2.

```python
d2y = 2 * (diff2 - diff1) / (X[i+2] - X[i])
```

### Что говорить на защите

- «Первая производная 1-го порядка — наклон секущей через 2 точки»
- «Первая производная 2-го порядка — используем 3 точки, точность выше»
- «Для равномерной сетки производная 2-го порядка = среднее левой и правой»
- «Вторая производная — удвоенная разделённая разность 2-го порядка»

## Задача 3.5 — Численное интегрирование (`task3_5.py`)

### Суть простыми словами

Нужно вычислить определённый интеграл ∫ f(x) dx. Берём два шага (h_1 = 1.0, h_2 = 0.5), считаем тремя методами, уточняем по Рунге-Ромбергу.

### Вариант 4

y = (3x + 4) / (2x + 7), пределы [-2, 2].

### Метод прямоугольников (формула 3.23)

Разбиваем отрезок на N частей. В каждой части берём значение функции **в середине** и умножаем на ширину:

F ≈ Σ h · f(x_{i-1/2})

```python
# Строки 39-43:
for i in range(N):
    x_mid = a + (i + 0.5) * h    # середина i-го отрезка
    result += h * y(x_mid)        # площадь прямоугольника
```

### Метод трапеций (формула 3.25)

Вместо прямоугольника — трапеция: берём значения на обоих концах отрезка:

F ≈ h · [y_0/2 + y_1 + y_2 + ... + y_{N-1} + y_N/2]

Концы делим пополам, потому что каждый из них входит только в одну трапецию.

```python
# Строки 52-57:
result = y(a)/2 + y(b)/2
for i in range(1, N):
    result += y(a + i*h)
result *= h
```

### Метод Симпсона (формула 3.28)

Вместо прямой на каждом отрезке — **парабола** через три точки (концы + середину). Самый точный из трёх.

F ≈ (h/6) · [y_0 + 4·y_{1/2} + 2·y_1 + 4·y_{3/2} + ... + y_N]

```python
# Строки 67-74:
result = y(a) + y(b)
for i in range(1, N):          # основные узлы — множитель 2
    result += 2 * y(a + i*h)
for i in range(N):             # полуузлы — множитель 4
    result += 4 * y(a + (i+0.5)*h)
result *= h / 6
```

### Уточнение по Рунге-Ромбергу (формула 3.30)

Идея: считаем интеграл с шагом h_1 и h_2 = h_1/2. Разница между ними позволяет **оценить и уменьшить** ошибку:

F ≈ F_{h2} + (F_{h2} - F_{h1}) / (k^p - 1)

где k = h_1/h_2 = 2, p — порядок метода (прямоугольники и трапеции: p=2, Симпсон: p=4).

```python
# Строки 83-84:
def runge_romberg(F_h, F_kh, k, p):
    return F_h + (F_h - F_kh) / (k**p - 1)
```

### Что говорить на защите

- «Три метода: прямоугольники (средние точки), трапеции (линейная аппроксимация), Симпсон (параболическая аппроксимация)»
- «Порядок точности: прямоугольники и трапеции — 2, Симпсон — 4»
- «Рунге-Ромберг: из двух приближений (h и h/2) вычисляем более точное»
- «Симпсон + Рунге-Ромберг даёт ошибку ~10^{-5} — лучший результат»
