# Лаба 4 — Обыкновенные дифференциальные уравнения (ОДУ)

## Общая идея лабы

Дифференциальное уравнение — это уравнение, в которое входит функция y(x) и её производные y', y''. Аналитически решить удаётся не всегда, поэтому нужны **численные методы**: шагаем по x маленькими шажками h и на каждом шаге приближённо вычисляем y.

- **Задача 4.1**: Задача Коши (знаем y и y' в начальной точке)
- **Задача 4.2**: Краевая задача (знаем y на обоих концах отрезка)

---

## Задача 4.1 — Задача Коши для ОДУ 2-го порядка (`task4_1.py`)

### Постановка (вариант 4)

Уравнение: **x^2·y'' - x(x^2-1)·y' - (x^2+1)·y = 0**

Начальные условия: y(1) = 1 + e^{1/2}, y'(1) = -1

Отрезок: x ∈ [1, 2], шаг h = 0.1

Точное решение: y(x) = (1 + e^{x^2/2}) / x

### Сведение к системе 1-го порядка

ОДУ 2-го порядка нельзя решать «напрямую» — сначала превращаем в **систему двух уравнений 1-го порядка**. Замена: z = y'.

Из уравнения выражаем y'':
y'' = [x(x^2-1)·z + (x^2+1)·y] / x^2

Получаем систему:
- y' = z (просто определение z)
- z' = [x(x^2-1)·z + (x^2+1)·y] / x^2

```python
# Строки 28-29:
def f_sys(x, y, z):   return z
def g_sys(x, y, z):   return (x*(x**2-1)*z + (x**2+1)*y) / x**2
```

### Метод Эйлера (формула 4.2)

Самый простой метод. Шагаем по касательной:

y_{k+1} = y_k + h · y'(x_k, y_k)

Для системы:
- y_{k+1} = y_k + h · f(x_k, y_k, z_k)
- z_{k+1} = z_k + h · g(x_k, y_k, z_k)

```python
# Строки 37-41:
y_new = y + h * f_sys(x, y, z)
z_new = z + h * g_sys(x, y, z)
```

**Точность**: O(h) — порядок 1. Каждый шаг вносит ошибку ~h^2, за N = (b-a)/h шагов накапливается O(h). Самый грубый метод.

### Метод Рунге-Кутты 4-го порядка (формула 4.16)

«Золотой стандарт» численного решения ОДУ. Вместо одного наклона (Эйлер) вычисляет **четыре** пробных наклона и берёт их взвешенную комбинацию.

Для каждого шага вычисляем 4 коэффициента K и L:
```
K1 = h·f(x, y, z)                     L1 = h·g(x, y, z)
K2 = h·f(x+h/2, y+K1/2, z+L1/2)      L2 = h·g(x+h/2, y+K1/2, z+L1/2)
K3 = h·f(x+h/2, y+K2/2, z+L2/2)      L3 = h·g(x+h/2, y+K2/2, z+L2/2)
K4 = h·f(x+h, y+K3, z+L3)            L4 = h·g(x+h, y+K3, z+L3)
```

Потом:
```
y += (K1 + 2K2 + 2K3 + K4) / 6
z += (L1 + 2L2 + 2L3 + L4) / 6
```

- K1, L1 — наклон в начале шага
- K2, L2 — наклон в середине (по первому пробному шагу)
- K3, L3 — наклон в середине (по второму пробному шагу)
- K4, L4 — наклон в конце шага

**Точность**: O(h^4). На порядки лучше Эйлера!

### Метод Адамса 4-го порядка (Адамс-Башфорт)

**Многошаговый** метод: использует значения в **4 предыдущих точках** для предсказания следующей. Экстраполяционная формула:

y_{k+1} = y_k + (h/24)·(55·f_k - 59·f_{k-1} + 37·f_{k-2} - 9·f_{k-3})

**Проблема**: нужны 4 начальных точки, а у нас только одна. Решение: первые 4 точки вычисляем **методом РК4** («разгон»).

```python
# Строки 69-70 — разгон: первые 4 точки из РК4
xs_rk, ys_rk, zs_rk = rk4(x0, y0, z0, h, min(3, n_steps))

# Строки 73-74 — основная формула:
y_new = ys[k] + h/24 * (55*fs[k] - 59*fs[k-1] + 37*fs[k-2] - 9*fs[k-3])
```

**Точность**: O(h^4) — как РК4, но на каждом шаге нужно **одно** вычисление f (вместо четырёх у РК4). Однако из-за экстраполяции ошибка чуть больше.

### Оценка по Рунге-Ромбергу

Считаем решение с шагом h и h/2. Разница оценивает ошибку:

ε ≈ |y_h(x_k) - y_{h/2}(x_k)| / (2^p - 1)

где p — порядок метода (Эйлер: p=1, РК4: p=4, Адамс: p=4).

### Результаты (ошибка в точке x=2)

| Метод | Ошибка | Порядок |
|-------|--------|---------|
| Эйлер | 3.67e-01 | O(h) — грубо |
| РК4 | 8.06e-06 | O(h^4) — отлично |
| Адамс | 5.16e-03 | O(h^4) — хуже РК4 из-за экстраполяции |

### Замечание об y'(1)

В PDF указано y'(1) = 2e^{1/2} - 1 ≈ 2.30, но **точное решение** даёт y'(1) = -1. Код использует корректное значение -1 (см. комментарий в файле).

## Задача 4.2 — Краевая задача (`task4_2.py`)

### Постановка (вариант 4)

Уравнение: **x^2(x+1)·y'' - 2y = 0**

Граничные условия: y(1) = 1 + 4·ln(2), y(2) = y_exact(2) ≈ 3.296

Точное решение (из PDF): y(x) = -1 + 2/x + 2(x+1)/x · ln|x+1|

### Отличие от задачи 4.1

В задаче 4.1 (Коши) мы знаем y и y' **в одной точке** (левом конце) и шагаем вправо. Здесь мы знаем y на **обоих концах**, но не знаем y'(a). Это принципиально другая задача.

### Нормальный вид ОДУ

y'' = 2y / [x^2·(x+1)]

Это ОДУ без y' (p(x)=0), с q(x) = 2/[x^2(x+1)], f(x) = 0.

### Метод 1: Метод стрельбы

**Идея (аналогия)**: представь, что ты стоишь на краю оврага (x=a) и стреляешь из пушки. Тебе нужно попасть в мишень на другом краю (x=b). Ты не знаешь, под каким углом стрелять (y'(a) = ?). Делаешь два пробных выстрела, смотришь куда попал, и по разнице **вычисляешь точный угол**.

1. **Выстрел 1**: пускаем y'(a) = η_1 = 0. Решаем задачу Коши (РК4) до x=b. Получаем y(b) = y_{b1}.
2. **Выстрел 2**: пускаем y'(a) = η_2 = 1. Снова решаем до x=b. Получаем y(b) = y_{b2}.
3. **Интерполяция**: для линейного ОДУ зависимость y(b) от η линейна. Поэтому:

η* = η_1 + (y_b - y_{b1}) · (η_2 - η_1) / (y_{b2} - y_{b1})

4. **Финальный выстрел**: решаем задачу Коши с η*.

```python
# Строки 67-76:
eta1 = 0.0
_, ys1, _ = rk4_ivp(a, ya, eta1, h, n)  # выстрел 1
yb1 = ys1[-1]

eta2 = 1.0

eta_star = eta1 + (yb - yb1) * (eta2 - eta1) / (yb2 - yb1)  # интерполяция

```

### Метод 2: Конечно-разностный метод

**Идея**: заменяем производные **конечными разностями** и получаем систему линейных уравнений.

Аппроксимация y'' центральными разностями:

y'' ≈ (y_{i-1} - 2·y_i + y_{i+1}) / h^2

Подставляем в ОДУ y'' = q(x)·y:

(y_{i-1} - 2·y_i + y_{i+1}) / h^2 = q_i · y_i

Перегруппируем:

y_{i-1} + (-2 - h^2·q_i)·y_i + y_{i+1} = 0

Это **трёхдиагональная СЛАУ** для y_1, y_2, ..., y_{N-1} (y_0 и y_N известны из граничных условий). Решаем прогонкой!

```python
# Строки 100-114 — формирование системы:
A[j] = 1.0                       # коэффициент при y_{i-1}
B[j] = -2.0 - h**2 * qi          # коэффициент при y_i
C[j] = 1.0                       # коэффициент при y_{i+1}
D[j] = 0.0                       # правая часть

# Строки 116-118 — учёт граничных условий:
D[0] -= A[0] * ya                # y_0 = ya переносим в правую часть
D[m-1] -= C[m-1] * yb            # y_N = yb переносим в правую часть
```

### Оценка погрешности

- **По Рунге-Ромбергу**: считаем с h и h/2. Стрельба (РК4 внутри, p=4): ε ~ 10^{-7}. Конечные разности (p=2): ε ~ 10^{-4}.
- **Сравнение с «точным»**: ~ 2·10^{-2}. Эта ошибка обусловлена не методом, а тем, что «точное» решение из PDF содержит неточность (см. замечание ниже).

### Замечание об ошибке в PDF

«Точное» решение y(x) = -1+2/x+2(x+1)/x·ln|x+1| **не удовлетворяет** уравнению x^2(x+1)y'' - 2y = 0. Подстановка даёт остаток -2(x-1), а не 0. Также граничное условие y(2) = -1+3·ln(2) ≈ 1.08 не совпадает с y_exact(2) = 3·ln(3) ≈ 3.30. Это ошибка в методичке. Код использует y_exact(2) для согласованности.

Погрешность Рунге-Ромберга (10^{-7} для стрельбы) подтверждает, что **метод работает правильно** — расхождение с «точным» решением обусловлено ошибкой в самом «точном» решении.
